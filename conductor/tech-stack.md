# Tech Stack: Voice & Vision Assistant for Blind

## 1. Core Languages & Frameworks
- **Python 3.11:** Primary programming language used across the application.
- **FastAPI:** Modern, fast web framework used for building the REST APIs (`api/server.py`).
- **Uvicorn:** ASGI web server used to run the FastAPI application.
- **LiveKit Agents SDK:** Used to build the real-time, low-latency agent coordinate loop (`AllyVisionAgent`).

## 2. External Cloud Services & APIs
- **LiveKit:** WebRTC SFU for handling real-time audio and video streams.
- **Deepgram:** Cloud API used for primary Speech-to-Text (STT) transcription with a 100ms SLA.
- **ElevenLabs:** Cloud API used for primary Text-to-Speech (TTS) generation with a 100ms SLA.
- **DuckDuckGo API:** Used for internet search capabilities.
- **Tavus:** Optional API used for generating a virtual avatar.

## 3. AI, Machine Learning & Computer Vision (Local & Cloud)
- **Ollama:** Used for serving local Large Language Models and Vision-Language Models (specifically running `qwen3.5:397b-cloud`). Also used for embeddings.
- **YOLO (`yolov8n.onnx`):** Local ONNX model used for real-time object detection and spatial awareness.
- **MiDaS (`midas_v21_small_256.onnx`):** Local ONNX model used for depth estimation.
- **OCR Stack:** EasyOCR, Tesseract, and MSER pipelines for extracting text from images.
- **Face Recognition Stack:** RetinaFace and MTCNN for consent-gated face detection and recognition.

## 4. Audio & Speech Processing (Fallbacks)
- **Local Edge TTS:** Primary fallback if ElevenLabs TTS fails.
- **pyttsx3:** Secondary, offline text-to-speech fallback if Edge TTS fails.
- **Whisper:** Local speech-to-text fallback if Deepgram STT fails.

## 5. Data Storage & Memory
- **SQLite:** Lightweight relational database for persistent storage, caching, and session data.
- **FAISS (Facebook AI Similarity Search):** Vector database used for Retrieval-Augmented Generation (RAG) and privacy-first memory indexing.

## 6. Observability, Monitoring & Infrastructure
- **Docker:** Containerization using `python:3.11-slim` base image.
- **Prometheus:** Used for scraping application and system metrics.
- **Grafana:** Used for visualizing metrics and monitoring SLOs/health.
- **Loki:** Used for pushing and querying structured JSON logs.
- **Alertmanager:** Used for handling alerts generated by Prometheus.

## 7. Network Protocols
- **WebRTC:** For real-time audio/video streaming between the user and LiveKit.
- **REST / HTTP/HTTPS:** For standard API communication and search functions.
- **gRPC:** Used specifically for the LiveKit-to-Deepgram STT plugin.
- **WebSocket / SSE:** For streaming LLM responses and real-time avatar data.
