A.2  KEY ALGORITHMS AND IMPLEMENTATION DETAILS
================================================================================

A.2.1  YOLOv8n ONNX Inference (core/vision/spatial.py)
------------------------------------------------------
- Model: Ultralytics YOLOv8n exported to ONNX format
- Input tensor: (1, 3, 640, 640), float32, normalized [0, 1]
- Output tensor: (1, 84, 8400) — 80 class scores + 4 box coordinates per anchor
- Preprocessing: Letterbox resize to 640×640, BGR→RGB, CHW transpose
- Post-processing: Confidence threshold 0.25, greedy NMS with IoU > 0.45
- Maximum detections capped at MAX_DETECTIONS = 2 for real-time performance
- Fallback: MockObjectDetector returns empty detections for testing

A.2.2  MiDaS v2.1 Depth Estimation (core/vision/spatial.py)
------------------------------------------------------------
- Model: Intel ISL MiDaS v2.1 small, ONNX format
- Input: (1, 3, 256, 256), ImageNet normalization (mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])
- Output: (1, 256, 256) inverse relative depth map
- Post-processing: Invert values, normalize, clip to [0.5, 10.0] meters pseudo-metric
- Downscale factor: DEPTH_DOWNSCALE = 4 for efficiency
- Fallback: SimpleDepthEstimator uses bbox-size heuristic when ONNX unavailable

A.2.3  Edge-Aware Segmentation (core/vision/spatial.py)
-------------------------------------------------------
- Algorithm: Sobel gradient computation on grayscale input
- Downscale to MAX_MASK_SIZE = (160, 120) for real-time processing
- Otsu automatic thresholding on gradient magnitude
- Boundary confidence = mean gradient along detected edges
- Edge pixels extracted from thresholded gradient map
- Per-detection binary mask extracted from bbox region

A.2.4  Spatial Fusion (core/vision/spatial.py)
----------------------------------------------
- FOV: 70° horizontal field of view
- Direction mapping: 7 bins (FAR_LEFT, LEFT, SLIGHT_LEFT, CENTER, SLIGHT_RIGHT, RIGHT, FAR_RIGHT)
  Thresholds: ±5° center dead zone, ±15° slight, ±25° moderate, beyond = far
- Priority classification:
  - CRITICAL:     distance < 1.0 m
  - NEAR_HAZARD:  1.0 m ≤ distance < 2.0 m  
  - FAR_HAZARD:   2.0 m ≤ distance < 5.0 m
  - SAFE:         distance ≥ 5.0 m
- Action recommendation: Direction-aware suggestion strings
  Example: "Stop! Chair half meter ahead" → for CRITICAL + CENTER

A.2.5  Risk Scoring (core/vqa/priority_scene.py)
-------------------------------------------------
- Formula:  R = 0.35·f_dist + 0.25·f_dir + 0.15·confidence + 0.25·f_collision
  - f_dist: inverse distance function (closer = higher score)
  - f_dir: centrality bias (center obstacles score higher)
  - confidence: raw detection confidence from YOLO
  - f_collision: estimated collision probability from trajectory analysis
- Returns top-3 hazards sorted by descending risk score
- Each hazard annotated with navigation cue text

A.2.6  OCR Pipeline (core/ocr/__init__.py)
-------------------------------------------
- Preprocessing: CLAHE contrast enhancement → bilateral denoising → Hough deskew
- Engine 1: EasyOCR (GPU-accelerated, multiple scripts)
- Engine 2: Tesseract fallback (when EasyOCR unavailable)
- Engine 3: MSER region detection (last resort)
- Deduplication: Intersection-over-Union (IoU) > 0.5 suppression
- Auto-probed at startup; engine availability cached

A.2.7  Memory and RAG (core/memory/__init__.py)
------------------------------------------------
- FAISS index: 384-dimensional embedding vectors (qwen3-embedding:4b)
- SQLite: Structured conversation logs, user preferences, face associations
- RAG pipeline: Query → FAISS top-k retrieval → context injection → LLM response
- Memory opt-in only (MEMORY_ENABLED default false)
- Backup/restore: periodic snapshots to data/memory_backup/

A.2.8  Voice Router (core/speech/voice_router.py)
--------------------------------------------------
- 17 intent types recognized via regex pattern matching
- Intent examples: DESCRIBE_SCENE, READ_TEXT, IDENTIFY_PERSON, NAVIGATE,
  FIND_OBJECT, CHECK_TIME, READ_BRAILLE, SCAN_QR, etc.
- Priority routing: Safety intents preempt informational queries
- Quick answers for time, date, and system status (<10ms)

A.5  ENVIRONMENT SETUP
================================================================================

System Requirements:
  - Python 3.10+
  - NVIDIA GPU with CUDA 11.8+ (RTX 4060 recommended, 8 GB VRAM)
  - 16 GB RAM minimum
  - Ubuntu 22.04 / Windows 11

Installation Steps:
  1. Clone repository:
     git clone https://github.com/<org>/Voice-Vision-Assistant-for-Blind.git
     cd Voice-Vision-Assistant-for-Blind

  2. Create virtual environment:
     python -m venv .venv
     source .venv/bin/activate        # Linux/macOS
     .venv\Scripts\activate           # Windows

  3. Install dependencies:
     pip install -e ".[dev]"
     pip install -r requirements.txt
     pip install -r requirements-extras.txt  # Optional

  4. Download models:
     python scripts/download_models.py

  5. Configure environment:
     cp .env.example .env
     # Edit .env with API keys (Deepgram, ElevenLabs, etc.)

  6. Start services:
     # Start Ollama (required for VQA and embeddings)
     ollama serve
     ollama pull qwen3-embedding:4b
     ollama pull qwen3-vl

  7. Run application:
     # LiveKit WebRTC agent:
     python -m apps.realtime.agent
     # FastAPI REST server:
     python -m apps.api.server

A.8  KNOWN LIMITATIONS AND WORKAROUNDS
================================================================================

1. MAX_DETECTIONS=2 limits scene complexity but ensures <100ms latency.
   Workaround: Increase for non-real-time batch processing.

2. MiDaS v2.1 small provides relative (not metric) depth.
   Workaround: Calibrated pseudo-metric mapping with [0.5, 10.0]m range.

3. Edge-aware segmentation uses Sobel gradients (not learned segmentation).
   Workaround: SAM or DeepLabV3+ integration planned for v2.

4. Face recognition requires opt-in consent and encrypted storage.
   Workaround: Social cue analyzer works without stored identities.

5. VQA response quality depends on Ollama model availability.
   Workaround: SiliconFlow cloud fallback automatically engaged.

6. OCR deskew uses Hough transform (fragile on curved surfaces).
   Workaround: TrOCR or PaddleOCR integration planned.
