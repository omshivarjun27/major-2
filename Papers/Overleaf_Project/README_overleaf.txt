QUICK START
-----------
1. Upload the entire Overleaf_Project/ folder to Overleaf (New Project → Upload).
2. Set main.tex as the main document.
3. Compiler: pdflatex (recommended) or xelatex.
4. Bibliography: bibtex (run pdflatex → bibtex → pdflatex → pdflatex).
5. Expected output: ~12-14 page IEEE two-column conference paper.

GENERATING FIGURES
------------------
1. Open Papers/diagram_prompts.txt
2. Copy each prompt into Paper Banana / Nano Banana Pro
3. Export each diagram as PNG (3000×2000, white background)
4. Replace the placeholder .txt files in figures/ with the actual .png files
5. Ensure filenames match exactly: fig1_architecture.png, fig2_dataflow.png, etc.


COMPILING LOCALLY
-----------------
If you prefer local compilation instead of Overleaf:

  # Install TeX Live (full) or MiKTeX
  cd Overleaf_Project/
  pdflatex main.tex
  bibtex main
  pdflatex main.tex
  pdflatex main.tex

Required LaTeX packages (all in standard distributions):
  - IEEEtran (document class)
  - cite, amsmath, amssymb, graphicx
  - url, hyperref, booktabs
  - algorithm, algorithmic
  - xcolor, subcaption, multirow


EDITING GUIDE
-------------
- Each section is in its own file under sections/. Edit independently.
- All citations reference keys in refs.bib. Use \cite{key} format.
- Tables are defined inline in results.tex. Modify data there directly.
- The risk formula in methods.tex uses equation environment — edit carefully.
- Author names and affiliations are in main.tex lines 15-25.
